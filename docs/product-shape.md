# Product Shape

## Product Definition

HotLoop is a local-first agent cockpit and toolroom for high-velocity AI and technology publishing.

It is not a standalone product and not a generic AI writing tool. The executor is still the agent. HotLoop is the operation surface and durable tool layer that lets the agent and human inspect what is worth writing now, preserve the evidence chain, produce reviewable article packages, render finished artifacts, and prepare publishing drafts.

## Working Name

HotLoop

The name should stay broad enough to cover radar, evidence, writing, rendering, and publishing. It should not be narrowed to one platform such as WeChat or one source such as X.

## First User

The first user is a solo AI / technology content creator who cares about:

- near-real-time AI product and research signals
- X explosive posts and product-lead comments
- GitHub Trending and open-source velocity
- international AI community discussion
- official updates from companies such as OpenAI, Anthropic, Google, GitHub, and Vercel
- producing evidence-backed WeChat-ready articles without manual file chaos

The first version is built for one power user working with an agent, not for teams or anonymous end users.

## Core Job

Help the creator answer:

```text
What is worth writing right now, why is it worth writing, what can be stated as fact, and how do we turn it into a finished artifact?
```

## Product Principles

1. Local-first before SaaS.
2. Content vault before database.
3. Agent-readable policy before hidden backend config.
4. Durable run state before conversational memory.
5. Modular radar and adapters before hardcoded sources.
6. Human-reviewed publishing before auto-publish.
7. Agent cockpit before standalone app.
8. Tools and durable state before autonomous backend replacement.

## First Workflow

```text
scan fresh sources
  -> classify candidates into P0-P4 lanes
  -> select topic
  -> collect evidence
  -> build public analysis
  -> draft article
  -> polish style and fact boundaries
  -> generate images
  -> render final HTML
  -> optionally create a platform draft
```

## Required Product Pages

1. Hotspot Radar
   - P0 X explosive posts
   - P1 GitHub / open-source trends
   - P2 international community hotspots
   - P3 official product updates
   - P4 paper signals

2. Topic Workspace
   - source summary
   - heat and freshness
   - evidence chain
   - facts / disputes / unknowns
   - writing angles
   - risk notes

3. Article Factory
   - article package status
   - draft text
   - image plan
   - image generation status
   - quality gates

4. Artifact Library
   - date-based final artifacts
   - clean final HTML list
   - preview and validation

5. Publish Console
   - platform compatibility check
   - image upload mapping
   - draft creation
   - publish-result record

## Non-goals

- Do not build a generic AI writer.
- Do not build a multi-user SaaS first.
- Do not make HotLoop the executor; the agent remains the executor.
- Do not migrate the content vault into a database first.
- Do not make a traditional CMS the primary product metaphor.
- Do not auto-publish to external platforms.
- Do not use opaque "agent search" as the main sourcing method.
- Do not hide source evidence behind generated summaries.

## Differentiation

Generic AI writing tools help turn a prompt into prose.

HotLoop helps decide what should be written, proves what can be safely written, produces the artifact, and keeps the workflow recoverable.
